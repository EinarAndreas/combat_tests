Sanity check for ComBat in BladderBatch tutorial.

2014-03-13 00:44:09

Overview

This small report aim to show that the application of ComBat as described in the user guide accompanying the R package sva presented in Leek et al. will lead to false results even for random data.

Two sanity checks will be performed

  • Swapping real data with random numbers
  • Permute the group labels whitinh batches Except for the above alterations, the batch adjustment and analysis are performed as described in the BladderBatch tutorial with additional plots to assess the results.

Permuted Group Labels

source("../../commonscripts/helper_functions.r")
nshuffleddatasets = 10
datamatrices_permuted = list()
labelschanged = vector()
permuted_pvalcounts = list()
for (i in 1:nshuffleddatasets) {
    permutedlabels = shufflesamplesinbatch(rownames(pheno), pheno$batch, pheno$cancer)
    labelschanged = c(labelschanged, sum(pheno$cancer != pheno[permutedlabels, 
        "cancer"]))
    permuteddata = exprs(bladderEset)[, permutedlabels]
    adjusteddata = as.matrix(ComBat(dat = permuteddata, batch = pheno$batch, 
        mod = model.matrix(~as.factor(pheno$cancer)), numCovs = NULL, par.prior = TRUE, 
        prior.plots = FALSE))
    this_p = f.pvalue(adjusteddata, mod, mod0)
    permuted_pvalcounts[[i]] = hist(this_p, plot = FALSE, breaks = 100)$counts
}
